# Format Adaptation Matrix

How content adapts across LinkedIn, Twitter/X, Newsletter, and Carousel.

---

## Core Rule: RE-CREATE, Don't Copy-Paste

Each platform has its own native language, rhythm, and audience mindset. The same insight must be rebuilt for each format. If you can copy-paste between platforms without changing anything, you haven't adapted it.

---

## Hook Adaptation by Format

| Hook Type | LinkedIn (800-1200 chars) | Twitter (280 chars) | Newsletter (subject line) | Carousel (cover slide) |
|-----------|--------------------------|--------------------|--------------------------|-----------------------|
| **Stat Drop** | Full stat + 1-2 sentences of context | Number + one sharp sentence + "Thread:" | Number in subject line, context in preview | Big number as visual focal point |
| **Scene Drop** | 2-3 lines of dialogue/action | One line of dialogue + thread indicator | Story tease: "A [person] said something that changed..." | Quote text on cover slide |
| **Name + Claim** | Company/person + bold observation + build | Brand + observation + "X things:" | Name in subject: "[Company] is doing something most won't" | Company logo/name + claim text |
| **Before/After** | Full contrast with context | Two versions separated by → | Before in subject, after in preview | Split slide: left vs right |
| **How-To Claim** | Promise + rejection of common approach + build | "How to X (not Y). Thread:" | "How to [outcome] without [common approach]" | "How to [X]" as title, "[Not Y]" as subtitle |

## Evidence Density by Format

| Format | Minimum Evidence Anchors | Notes |
|--------|-------------------------|-------|
| **LinkedIn** | 1 per post | One strong stat or case study. More can feel forced at 800-1200 chars. |
| **Twitter** | 2-3 across thread | Spread across body tweets. Each evidence tweet is screenshot-worthy. |
| **Newsletter** | 3+ with cross-references | Highest density. Include newsletter source attributions. |
| **Carousel** | 1 per 2-3 slides | Evidence per slide when possible, but readability > density. |

## What to CUT When Going Long → Short

When compressing a newsletter into shorter formats:

| Cut This | Why |
|----------|-----|
| Context paragraphs | Shorter formats start at the point |
| Counterarguments | Save for newsletter depth |
| Setup/backstory | LinkedIn needs 1 sentence of setup max |
| Multiple evidence anchors | Pick the single strongest one |
| Practitioner callout section | Too detailed for short form |
| Transition sentences | Short formats use white space instead |
| "What most get wrong" sections | Compress to 1 rejection-based line |

**Compression priority:** Keep the sharpest insight + strongest evidence + most specific example. Cut everything that supports but doesn't carry.

---

## What to ADD When Going Short → Long

When expanding a LinkedIn post into a newsletter:

| Add This | Why |
|----------|-----|
| Opening story (200-400 words) | Newsletters earn attention through narrative |
| Deeper framework exposition | Show the full model, not just the conclusion |
| Counterarguments | Address "but what about..." proactively |
| Additional evidence (2-3 more anchors) | Newsletter readers expect proof density |
| Cross-newsletter references | Shows breadth of reading |
| Practitioner callout section | "What most get wrong" — operator credibility |
| Personal case study or teardown | The "only I could have written this" section |
| P.S. with secondary CTA | Newsletter-specific element |

**Expansion priority:** Don't pad. Every added section must introduce new information, new evidence, or new perspective. If a section just restates what the LinkedIn post said in more words, cut it.

## Platform Voice Notes

Same Sourav voice across all platforms. The adjustment is RHYTHM, not personality.

| Platform | Rhythm | Sentence Style |
|----------|--------|---------------|
| **LinkedIn** | Punchy paragraphs, 1-2 sentences each, white space between | Fragments OK. Short declarative. → arrows. |
| **Twitter** | One idea per tweet, vary short/long tweets, line breaks within | Even punchier. One-liners hit. Questions work in body (not hooks). |
| **Newsletter** | Mix short + long paragraphs, section headers, flowing narrative | Can use longer sentences. Narrative connectors OK. Still no filler. |
| **Carousel** | One point per slide, max 3-4 lines, visual hierarchy | Shortest text. Headlines + 1-2 supporting lines. |

## Angle Adaptation by Format

### Angle 1: Opinionated Reframe
- **LinkedIn:** Full build — hook → setup → turn → evidence → landing
- **Twitter:** Hook tweet is the claim. Thread body is the evidence. Last tweet is the reframe.
- **Newsletter:** Extended build with counterarguments addressed, multiple evidence anchors, practitioner callout
- **Carousel:** Myth → Reality per slide structure

### Angle 2: Framework / Mental Model
- **LinkedIn:** Compact framework — numbered steps, 1 line per step + 1 line rationale
- **Twitter:** Framework name in hook. One step per tweet. Application in close.
- **Newsletter:** Full framework exposition with context, examples per step, common mistakes
- **Carousel:** One step per slide. Visual progression. Application on penultimate slide.

### Angle 3: Story-Led with Earned Insight
- **LinkedIn:** Tight story — hook drops into moment, 3-4 sentences of story, observation, insight, landing
- **Twitter:** Story arc across 7-10 tweets. Setup → tension → turning point → lesson.
- **Newsletter:** Full Epiphany Bridge (7-step). Deepest narrative. 200+ word story section.
- **Carousel:** Key moments as slides. Quote/dialogue on individual slides. Lesson on close slide.

## Distribution Workflow

### Pillar → Micro (Default: Newsletter First)
1. Generate newsletter article (pillar) — deepest research, full evidence
2. Extract core insight, best evidence anchor, strongest framework element
3. Generate LinkedIn post — compress to 800-1200 chars, pick best angle
4. Generate Twitter thread — rebuild for tweet-sized ideas, 7-10 tweets
5. Generate carousel brief — extract framework/progression, 5-10 slides
6. Run each through Rubber Duck Escalator (format-adapted scoring)

### Micro → Pillar (Reverse: LinkedIn → Newsletter)
1. Read existing LinkedIn post
2. Identify compressed elements — implied evidence, hinted stories, mentioned frameworks
3. Mini-research: load Brain sections for deeper evidence
4. Expand using newsletter structure — full Epiphany Bridge for stories, extended framework for models
5. Generate subject line + preview text
6. Run through Rubber Duck Escalator (newsletter-adapted scoring)
